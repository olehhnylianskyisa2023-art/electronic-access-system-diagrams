# UML Діаграми - Електронна система пропуску в гуртожиток 🏢

Цей репозиторій містить комплект професійних UML-діаграм для системи "Електронна система пропуску в гуртожиток" з розпізнаванням обличчя.

## 📊 Список діаграм

### 1️⃣ Діаграма варіантів використання (Use Case Diagram)
**Файл:** `01-use-case-diagram.puml`

Описує всіх акторів системи та їхню взаємодію:
- **Актори:** Студент, Охоронець, Адміністратор, Система розпізнавання обличчя
- **Варіанти використання:** Вхід, Реєстрація обличчя, Перегляд логів, Управління доступом, Сповіщення

---

### 2️⃣ Діаграма класів (Class Diagram)
**Файл:** `02-class-diagram.puml`

Структура даних та класи системи:
- **Класи:** User, Student, Security, Administrator, FaceRecognition, AccessLog, Room, Building, Alert
- **Відношення:** Спадкування, асоціація, композиція
- **Атрибути та методи** для кожного класу

---

### 3️⃣ Діаграма кооперації (Collaboration Diagram)
**Файл:** `03-collaboration-diagram.puml`

Взаємодія компонентів під час процесу входження:
- Студент → Камера → Розпізнавання → База даних
- Контролер доступу → Двері
- Логування в систему

---

### 4️⃣ Діаграма послідовності (Sequence Diagram)
**Файл:** `04-sequence-diagram.puml`

Часова послідовність взаємодій при успішному входженні:
1. Захоплення зображення обличчя
2. Передача в систему розпізнавання
3. Порівняння з базою даних
4. Видання дозволу/заборони
5. Запис логу

---

### 5️⃣ Діаграма станів (State Diagram)
**Файл:** `05-state-diagram.puml`

Стани системи контролю доступу:
- **Очікування** → **Сканування** → **Обробка**
- Переходи в стани: Дозвіл, Заборона, Тривога
- Повернення до очікування

---

### 6️⃣ Діаграма діяльності (Activity Diagram)
**Файл:** `06-activity-diagram.puml`

Процес входження з прийняттям рішень:
- Захоплення обличчя
- Розпізнавання
- Перевірка доступу
- Видання статусу (дозвіл/заборона)
- Логування

---

### 7️⃣ Діаграма компонентів (Component Diagram)
**Файл:** `07-component-diagram.puml`

Архітектура системи та залежності:
- **Клієнтська частина:** Web UI, Мобільний додаток
- **Серверна частина:** API Gateway, Recognition Service, Auth Service, Logging Service
- **Зберігання:** База даних, Кеш
- **Зовнішні системи:** Камери, Контролери входу

---

### 8️⃣ Діаграма розгортання (Deployment Diagram)
**Файл:** `08-deployment-diagram.puml`

Розподіл компонентів на фізичні вузли:
- **Гуртожиток:** Камери, контролери входу
- **Сервер:** API, розпізнавання, логування
- **База даних:** PostgreSQL, Бекап
- **Адміністративна станція:** Панель управління
- **Мобільні пристрої:** Додаток для студентів

---

### 9️⃣ Розширена діаграма діяльності (Extended Activity Diagram)
**Файл:** `09-extended-activity-diagram.puml`

Паралельні процеси системи:
1. **Контроль входження** - Розпізнавання та видання доступу
2. **Логування** - Запис всіх операцій
3. **Моніторинг** - Перевірка тривоги та сповіщення

---

## 🚀 Як використовувати

### Варіант 1: PlantUML Online Editor
1. Перейдіть на https://www.plantuml.com/plantuml/uml/
2. Скопіюйте код діаграми (.puml файл)
3. Вставте в редактор
4. Завантажте як PNG/SVG

### Варіант 2: VS Code
1. Встановіть розширення PlantUML
2. Відкрийте .puml файл
3. Натисніть Alt+D для перегляду
4. Експортуйте діаграму

### Варіант 3: GitHub Markdown
Вставте код в README.md з мовою `plantuml` - GitHub автоматично рендеритиме!

## 💡 Система "Електронна система пропуску в гуртожиток"

**Основна функція:** Контроль входження в гуртожиток з використанням розпізнавання обличчя

**Компоненти:**
- 📸 Камери розпізнавання обличчя
- 🤖 AI-модель розпізнавання
- 🗄️ База даних студентів та доступу
- 🚪 Контролери електронних дверей
- 📱 Мобільний додаток для студентів
- 👨‍💼 Адміністративна панель

**Користувачі:**
- 👨‍🎓 Студенти гуртожитку
- 👮 Охоронці
- 👨‍💻 Адміністратори системи
